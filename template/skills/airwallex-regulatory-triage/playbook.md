# Triage Playbook

The decision rules. Reviewer-edited; loaded into every session.

## Flag if any of the following apply

1. **Direct licence touch.** The update changes rules for any licence type
   in `corpus/licences.yaml` in a jurisdiction Airwallex operates in.
2. **Capital / safeguarding.** Changes to safeguarding ratios, ring-fencing
   rules, or capital adequacy for payment institutions or EMIs.
3. **Cross-border flow.** Changes to outbound/inbound payment reporting,
   FX licensing, or correspondent-banking obligations.
4. **AML / sanctions.** New sanctioned entities, jurisdictions, or
   typologies that affect screening logic.
5. **Stablecoin / crypto rails.** Any rule touching fiat-backed stablecoins
   or VASP licensing in operating jurisdictions.
6. **Authorisation perimeter.** Changes to what activities require which
   licence (a frequent source of unexpected scope creep).

## Ignore if

- Update is a consultation paper without policy commitment (note in
  rationale, but `decision = "ignore"`).
- Jurisdiction not in `corpus/licences.yaml`.
- Update is a personnel announcement, speech, or press release without
  rule change.
- Identical update already memoed within the last 14 days.

## Severity hint (informational, does not change decision)

- **High**: enforcement action, immediate effective date, capital change.
- **Medium**: rule change with > 30 day implementation runway.
- **Low**: clarifying guidance, no operational change.

## Owner routing

| Affected licence range | Suggested owner            |
|------------------------|----------------------------|
| UK-*                   | Head of UK Compliance      |
| EU-*                   | Head of EU Compliance      |
| US-*                   | Head of US Compliance      |
| APAC-*                 | Head of APAC Compliance    |
| MULTI                  | Group GC                   |

When `affected_licences` spans regions, route to Group GC and CC the
regional owners.

## Reviewer feedback policy

When a memo is corrected, the diff is captured to
`out/feedback/<update_id>.diff`. The next run loads the 5 most recent
diffs as few-shot examples in the decision step. Do not edit this file
to encode lessons — let the diffs do that work, and review them
quarterly to promote stable patterns into rules above.
